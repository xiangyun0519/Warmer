## 1. 产品定位

| 项目 | 内容 |
|------|------|
| **名字** | 暖流 |
| **slogan** | 让每一段关系都被看见 / 记录每一段温度 |
| **定位** | 大学生关系维护工具（单机版MVP） |
| **目标用户** | 大学生（18-25岁） |
| **核心价值** | 帮助用户记录、维护重要的人际关系 |
| **技术方案** | 纯前端SPA + localStorage |

---

## 2. 核心功能

### 2.1 关系卡片

- **添加重要的人**：头像（emoji）、昵称、关系类型、关系等级、生日（可选）
- **关系类型**：朋友（friend）、恋人（lover）、家人（family）
- **关系等级**：陌生 → 认识 → 熟识 → 亲密 → 挚友

### 2.2 温度曲线

- **记录方式**：每周问用户"和XXX的关系最近好吗？"
- **选项**：变冷（-10）/ 没变化（0）/ 变暖（+10）
- **可视化**：折线图展示关系变化趋势
- **记录**：每次记录可附上简短的"今天发生啥了"

### 2.3 暖心建议

用户在打开App时看到针对性的维护建议。

## 4. 暖心建议规则（10+条）

| 场景 | 建议文案 |
|------|----------|
| 14天没联系+降温 | "你和{name}已经两周没联系了，发个消息吧" |
| 7天没联系 | "记得和{name}保持联系哦" |
| 持续降温 | "关系需要维护，找个机会聊聊？" |
| 持续升温 | "你们最近关系升温很快，可以约出来聚聚" |
| 新关系 | "刚认识{name}，可以主动打个招呼" |
| 生日当天 | "今天是{name}的生日，送个祝福吧" |
| 恋人+长期未更新 | "和{name}在一起很久了，记得制造小惊喜" |
| 家人+长期未更新 | "给家人打个电话吧，他们在想你" |
| 温度低于40 | "这段关系好像遇到了问题，也许需要主动修复" |
| 温度高于80 | "这段关系很健康，继续保持" |
| 关系刚建立 | "新朋友要好好维护哦，记得保持联系" |

## 5. 用户路径

```
用户打开App
    ↓
看到"关系温度首页"（按红黄绿排序的所有关系卡片）
    ↓
点击"+"添加一个人
    ↓
填写昵称、选择头像、选关系类型
    ↓
添加成功，回到首页
    ↓
点击某人的卡片
    ↓
看到关系详情页（温度曲线、记录、建议）
    ↓
每周回来更新温度 → 看到新的暖心建议
```
